Budget Application

User Experience, HTML/CSS, Django and Python

VIEW PROTOTYPE

Background

As part of my senior capstone at Weber State University, I worked with a group to design and create a budgeting application. The application needed to responsively function as both a desktop version and mobile version, as well as have a login/logout functionality to protect the privacy of the user. Additionally, the project needed to include a total budget, upcoming bills, and a daily spending rate calculated by the number of days in the current month. The project was coded using Django and Python, which has become my favorite coding medium.

Research Methods

My first responsibility of our team was to create a functional wireframe and prototype for our project. This would allow us to decide what pages we wanted, how we wanted to display our information, and more. It also allowed us to decide how we wanted the app to function between the mobile version versus desktop version; for example, the navbar at the bottom of the mobile version would adjust to the side of the screen when it reaches a desktop viewport.

The Project

The functions of this application in it's current state include:

  • Dashboard: displays various widgets with the most important information.
  • Wallet: displays total account balance, upcoming bills, and all transactions created under that account. Also gives the option to add, edit or delete transactions.
  • Spending: displays the daily spending rate (calculated by number of days in a month), the next payday, progress bar of available spending for the month, and dropdown menus to show transactions by category.
  • Reports: displays a chart with number of transactions and amount spent/earned for each category.
  • Profile: Only transactions that have been created under the active user will be used. They can change the next payday here.
